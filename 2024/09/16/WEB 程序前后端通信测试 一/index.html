WEB 程序前后端通信测试 一

前言

之前我们已经在本地配置好了 Apache 服务器。这是用来开发 WEB 程序的前期准备。本篇文章将测试前后端的连接测试。

准备

流程

开发一份 PHP 的测试代码，名为 aaa.php：

<?php
    // 允许跨域请求
    header("Access-Control-Allow-Origin: *");
    header("Access-Control-Allow-Methods: GET, POST, OPTIONS");
    header("Access-Control-Allow-Headers: Content-Type");
    
    // 检查请求方法
    if ($_SERVER['REQUEST_METHOD'] === 'POST') 
  {
        // 处理 POST 请求
        $data = $_POST['data']; // 获取 POST 数据
        echo "Received POST data: " . htmlspecialchars($data);
    } 
    elseif ($_SERVER['REQUEST_METHOD'] === 'GET') 
  {
        // 处理 GET 请求
        if (isset($_GET['data'])) {
            $data = $_GET['data']; // 获取 GET 数据
            echo "Received GET data: " . htmlspecialchars($data);
        } else {
            echo "No GET data received";
        }
    } 
    else 
  {
        echo "Unsupported request method";
    }
?>

把它放到 /opt/homebrew/var/www 这个路径中：

再开发一份HTML和JavaScript的测试网页：

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>PHP AJAX Test</title>
</head>
  
<body>

    <h1>Send GET or POST Request to PHP</h1>

    <!-- 穿过去的数据 -->
    <input type="text" id="dataInput" placeholder="Enter some text">
    <br><br>
      
      <!-- GET -->
    <button onclick="sendGetRequest()">Send GET Request</button>
    <!-- POST -->
    <button onclick="sendPostRequest()">Send POST Request</button>

      <!-- 返还服务器结果 -->
    <h2>Response from Server:</h2>
    <pre id="response"></pre>

  
    <script>
                
        // 发送GET请求（带着‘dataInput’的数据），返还结果到 ‘response’中
        function sendGetRequest() {
          const data = document.getElementById('dataInput').value;
          fetch(`http://localhost:8080/aaa.php?data=${encodeURIComponent(data)}`)
              .then(response => response.text())
              .then(text => {
                  document.getElementById('response').textContent = text;
              })
              .catch(error => console.error('Error with GET request:', error));
        }

        // 发送POST请求（带着‘dataInput’的数据），返还结果到 ‘response’中
        function sendPostRequest() {
            const data = document.getElementById('dataInput').value;
            fetch('http://localhost:8080/aaa.php', {
                method: 'POST',
                headers: {
                    'Content-Type': 'application/x-www-form-urlencoded',
                },
                body: `data=${encodeURIComponent(data)}`
            })
            .then(response => response.text())
            .then(text => {
                document.getElementById('response').textContent = text;
            })
            .catch(error => console.error('Error with POST request:', error));
        }
    </script>
</body>
  
</html>

拓展

因为 PHP 需要提前写好循序跨域请求：

     // 允许跨域请求
    header("Access-Control-Allow-Origin: *");
    header("Access-Control-Allow-Methods: GET, POST, OPTIONS");
    header("Access-Control-Allow-Headers: Content-Type");

所以如果 PHP 文件很多而且有不同服务器都会发来请求的话，重复写这个很麻烦可以直接配置 Apache 服务器，变成全局设置：

修改 Apache 的 .htaccess 或主配置文件（通常是 httpd.conf）：

在 .htaccess 或 Apache 配置文件中添加以下内容：

<IfModule mod_headers.c>
    Header set Access-Control-Allow-Origin "*"
    Header set Access-Control-Allow-Methods "GET, POST, OPTIONS"
    Header set Access-Control-Allow-Headers "Content-Type"
</IfModule>

删除 PHP 中的重复代码。

重启：

brew services restart httpd

然后再试一次。
