Mac 下载 Apache （不使用本机自带的 Apache）

前言

Mac 正常自带 Apache，但是并不是全功能的 Apache。例如 PHP 的配置在自带的 Apache 中就是不支持的。所以最好重新安装一个功能齐全的 Apache。

本文将使用 brew 安装 Apache HTTP Server。

提前

我们先来了解一下 Mac 本身自带的 Apache 的目录结构：

  • 配置文件 /etc/apache2/httpd.conf
  • 站点根目录 /Library/WebServer/Documents/
  • 日志路径 /var/log/apache2

但是我们取配置文件中确定一下是不是已经不支持PHP了：

xin@xindeMacBook-Pro apache2 % grep 'removed' httpd.conf
#PHP was deprecated in macOS 11 and removed from macOS 12
xin@xindeMacBook-Pro apache2 %

所以，如果我们依然要使用 PHP代码的话，我们只能重新安装 Apache。

安装流程

使用 homebrew 安装 httpd：

brew install httpd

等待一段时间安装好后最后的提示为：

Removing: /Users/xin/Library/Caches/Homebrew/pkg-config_bottle_manifest--0.29.2_3... (11.0KB)
Pruned 0 symbolic links and 8 directories from /opt/homebrew
==> Caveats
==> httpd
DocumentRoot is /opt/homebrew/var/www.

The default ports have been set in /opt/homebrew/etc/httpd/httpd.conf to 8080 and in
/opt/homebrew/etc/httpd/extra/httpd-ssl.conf to 8443 so that httpd can run without sudo.

To start httpd now and restart at login:
  brew services start httpd
Or, if you don't want/need a background service you can just run:
  /opt/homebrew/opt/httpd/bin/httpd -D FOREGROUND
==> php
To enable PHP in Apache add the following to httpd.conf and restart Apache:
    LoadModule php_module /opt/homebrew/opt/php/lib/httpd/modules/libphp.so

    <FilesMatch \.php$>
        SetHandler application/x-httpd-php
    </FilesMatch>

Finally, check DirectoryIndex includes index.php
    DirectoryIndex index.php index.html

The php.ini and php-fpm.ini file can be found in:
    /opt/homebrew/etc/php/8.3/

To restart php after an upgrade:
  brew services restart php
Or, if you don't want/need a background service you can just run:
  /opt/homebrew/opt/php/sbin/php-fpm --nodaemonize
xin@xindeMacBook-Pro ~ % brew services start httpd
==> Successfully started `httpd` (label: homebrew.mxcl.httpd)
xin@xindeMacBook-Pro ~ % cd /opt/homebrew/etc/httpd

如果有提示中的需求，可以按照提示的命令进行配置。

安装完成之后使用 brew services list 命令查看一下使用 Homebrew 安装并作为服务管理的所有程序的状态：

xin@xindeMacBook-Pro ~ % brew services list
Name    Status  User File
httpd   started xin  ~/Library/LaunchAgents/homebrew.mxcl.httpd.plist
php     started xin  ~/Library/LaunchAgents/homebrew.mxcl.php.plist
unbound none         
xin@xindeMacBook-Pro ~ %

也可以使用 which 和 where 命令检查一下服务器的路径：

xin@xindeMacBook-Pro ~ % which httpd
/usr/sbin/httpd
xin@xindeMacBook-Pro ~ % which apachectl
/usr/sbin/apachectl
xin@xindeMacBook-Pro ~ % where apachectl
/opt/homebrew/bin/apachectl
/usr/sbin/apachectl
xin@xindeMacBook-Pro ~ % where httpd
/opt/homebrew/bin/httpd
/usr/sbin/httpd
xin@xindeMacBook-Pro ~ %

小小总结一下，homebrew 安装的 httpd：

  • 配置文件 /opt/homebrew/etc/httpd/httpd.conf
  • 站点根目录 /opt/homebrew/var/www
  • 日志路径 /opt/homebrew/var/log/httpd

这个时候就下载完了，接下来需要在 Apache 的配置文件中添加 PHP 的配置：

如果 PHP 未安装，可以使用 Homebrew 安装 PHP：

brew install php

找到配置文件：

xin@xindeMacBook-Pro ~ % cd /opt/homebrew/etc/httpd            
xin@xindeMacBook-Pro httpd % ls
extra        httpd.conf    magic        mime.types    original
xin@xindeMacBook-Pro httpd % vim httpd.conf 
xin@xindeMacBook-Pro httpd %

接着，确保 Apache 的配置文件中已经启用了 PHP 模块。在 Apache 的配置文件 /usr/local/etc/httpd/httpd.conf 中，检查以下几行是否存在或取消注释（去掉前面的 #）：

LoadModule php_module /opt/homebrew/opt/php/lib/httpd/modules/libphp.so

加载了 PHP 模块，使 Apache 能够处理 PHP 文件。

这行代码的作用是加载 PHP 模块到 Apache 服务器中。

  • LoadModule 指令告诉 Apache 加载指定的模块。
  • php_module 是模块的名称，它使 Apache 能够识别并处理 PHP 文件。
  • /opt/homebrew/opt/php/lib/httpd/modules/libphp.so 是 PHP 模块的实际路径，它是通过 Homebrew 安装 PHP 后所在的位置。

这段配置用于指定如何处理以 .php 为后缀的文件。确保所有 .php 文件都由 PHP 模块处理，而不是直接作为静态文件返回给客户端。

<FilesMatch \.php$>
    SetHandler application/x-httpd-php
</FilesMatch>

指定了所有 .php 文件都应该通过 PHP 模块处理。

  • <FilesMatch .php$>：这一行使用正则表达式匹配所有以 .php 结尾的文件（.php$）。

  • SetHandler application/x-httpd-php：这一行指示 Apache 使用 application/x-httpd-php 处理器来处理这些匹配到的文件，也就是说，所有匹配 .php 的文件都将由 PHP 解析器处理。

还需要告诉 Apache 处理 PHP 文件。在 httpd.conf 文件中，添加以下内容（如果还没有）：

<IfModule php_module>
    AddType application/x-httpd-php .php
</IfModule>

确保在 PHP 模块加载时，Apache 会把 .php 文件当作 PHP 文件类型来处理。

  • ：这是一种保护措施，确保仅在 PHP 模块已经被加载时才应用此配置。它的意思是：“如果 PHP 模块存在，那么执行下面的配置”。
  • AddType application/x-httpd-php .php：这行指令告诉 Apache，任何以 .php 结尾的文件都应该被识别为 application/x-httpd-php 类型，并交由 PHP 模块处理。

运行尝试

配置完后就可以尝试启动服器了，以下是一些常用命令：

# 启动服务
sudo apachectl start

# 重启服务
sudo apachectl stop

# 关闭服务
sudo apachectl restart

# 关闭开机自动启动
sudo launchctl unload -w /System/Library/LaunchDaemons/org.apache.httpd.plist

# 开启开机自动启动
sudo launchctl load -w /System/Library/LaunchDaemons/org.apache.httpd.plist

或者，手动启动 Apache（不作为服务运行）：

httpd -k start

启动：

xin@xindeMacBook-Pro ~ % brew services start httpd
==> Successfully started `httpd` (label: homebrew.mxcl.httpd)
xin@xindeMacBook-Pro ~ %

我这个已经启动了，所以提示 already started。

重启：

xin@xindeMacBook-Pro ~ % brew services restart httpd
Stopping `httpd`... (might take a while)
==> Successfully stopped `httpd` (label: homebrew.mxcl.httpd)
==> Successfully started `httpd` (label: homebrew.mxcl.httpd)
xin@xindeMacBook-Pro ~ %

这个就是重启案例。

添加一个测试用的 PHP 工程：

xin@xindeMacBook-Pro ~ % cd /opt/homebrew/var/www/        
xin@xindeMacBook-Pro www % ls
aaa.php        cgi-bin        index.html
xin@xindeMacBook-Pro www % vim test.php
xin@xindeMacBook-Pro www %

test.php

<?php 
    phpinfo(); 
?>

创建好后重新启动一下服务器，然后取浏览器中访问：http://localhost:8080/test.php 查看效果：

成功了！
