前言

本篇文章将对后端代码进行进一步的开发。主要完成以下功能：

流程

PHP 文件架构

首先对后端架构进行整体设计，以下是项目文件结构：

project/
├── public/                    // 公共访问目录，入口文件
│   └── index.php
├── app/                       // 应用核心代码
│   ├── controllers/           // 控制器
│   │   └── UserController.php
│   ├── models/                // 数据模型
│   │   └── User.php
│   ├── core/                  // 核心类
│   │   ├── Router.php         // 路由类
│   │   ├── Request.php        // 请求类
│   │   ├── Response.php       // 响应类
│   │   ├── Controller.php     // 基础控制器
│   │   └── Model.php          // 基础模型
│   └── routes.php             // 路由定义文件
├── config/                    // 配置文件
│   └── config.php
├── vendor/                    // 第三方库（可选，用于自动加载）
│   └── autoload.php
├── logs/                      // 日志文件
├── tests/                     // 测试代码
└── composer.json              // Composer 配置文件（可选）

说明：

  • public/：存放入口文件 index.php，所有请求都将通过此文件。
  • app/：核心应用程序代码。
  • controllers/：控制器，处理请求并返回响应。
  • models/：数据模型，处理数据库交互和业务逻辑。
  • core/：核心类，包括路由、请求、响应、基础控制器和模型等。
  • routes.php：定义应用的路由规则。
  • config/：配置文件，如数据库连接信息等。
  • vendor/：使用 Composer 自动加载类（即使不使用第三方库，也建议使用 Composer 来管理自动加载）。
  • logs/：存放日志文件。
  • tests/：测试代码，确保代码质量。
  • composer.json：Composer 配置文件，用于管理依赖和自动加载。

程序架构

采用简化的 MVC（模型-视图-控制器）架构，实现代码的模块化和可维护性。

  • 路由器（Router）：解析请求的 URL 和方法，分发到相应的控制器和方法。

  • 控制器（Controller）：处理请求，调用模型获取数据，返回响应。

  • 模型（Model）：与数据库交互，处理业务逻辑。

  • 请求和响应（Request & Response）：封装 HTTP 请求和响应，方便数据的获取和返回。

运行逻辑和整体流程

请求流程

  1. 客户端请求：前端（Web、移动端等）向后端 API 发起 HTTP 请求，例如 GET /api/users/1。
  2. 入口文件：请求到达 public/index.php，作为应用的唯一入口。
  3. 自动加载：使用 Composer 的自动加载功能，引入所需的类。
  4. 初始化路由器：Router 类解析请求的路径和方法。
  5. 路由分发：根据路由规则，路由器调用对应的控制器方法。
  6. 控制器处理：控制器接收请求，调用模型获取或操作数据。
  7. 模型交互：模型与数据库交互，执行相应的查询或操作。
  8. 响应返回：控制器将数据封装为 JSON 格式，使用 Response 类返回给客户端。

请求类型示例

  • 浏览数据（GET）：GET /api/users，获取用户列表。
  • 获取单个数据（GET）：GET /api/users/{id}，获取指定用户信息。
  • 添加数据（POST）：POST /api/users，创建新用户。
  • 更新数据（PUT）：PUT /api/users/{id}，更新指定用户信息。
  • 删除数据（DELETE）：DELETE /api/users/{id}，删除指定用户。

代码规范和最佳实践

  • 遵循 PSR 标准：遵循 PHP-FIG 的 PSR-1 和 PSR-12 编码标准，保持代码风格一致。
  • 自动加载：使用 Composer 的 PSR-4 自动加载规范，方便类的加载和管理。
  • 命名空间：使用命名空间组织代码，与文件结构保持一致。
  • 错误处理：统一的错误和异常处理，返回有意义的 HTTP 状态码和错误信息。
  • 安全性：防范 SQL 注入、XSS 等常见安全问题，使用预处理语句和适当的输入验证。
  • 配置管理：将可变的配置项（如数据库连接信息）放在配置文件中，方便管理和维护。

留出可扩展空间

  • 模块化设计：代码按照功能模块组织，方便未来添加新功能或模块。
  • 中间件支持：设计路由器时考虑中间件机制，方便添加认证、日志等功能。
  • 环境配置：支持多环境配置（开发、测试、生产），方便应用部署和维护。
  • 日志记录：完善的日志机制，方便调试和错误追踪。
  • 单元测试：编写测试代码，确保代码质量和稳定性。
