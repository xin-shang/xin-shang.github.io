操作步骤

安装Hexo

在桌面右击，选择Git Bash Here 使用npm或者cnpm安装即可，安装命令如下：

安装命令行工具

//安装hexo博客
$ npm install hexo-cli -g
$ npm install hexo --save

//安装插件（插件有很多，这个是用来将内容用git部署的插件）
$ npm install --save hexo-deployer-git

可以用如下代码确定node & npm 是否存在以及它们的版本

创建一个新建文件夹，这里将会成为放置博客内容的地方。

文件的名字随意。我现在是“blog”

Hexo初始化配置：

$ hexo init
INFO  Cloning hexo-starter https://github.com/hexojs/hexo-starter.git
[32mINFO [39m Install dependencies
added 182 packages from 420 contributors and audited 188 packages in 6.253s

12 packages are looking for funding
  run `npm fund` for details

found 0 vulnerabilities

INFO  Start blogging with Hexo!

$ hexo init 全名：hexo initialization， 用来表示初始化hexo博客的意思。

然后可以用 dir & ls -s 来看看生成了哪些文件。

这时用：

$ hexo clean
INFO  Validating config
INFO  Deleted database.
INFO  Deleted public folder.


$ hexo s
INFO  Validating config
INFO  Start processing
INFO  Hexo is running at http://localhost:4000 . Press Ctrl+C to stop.

$ hexo s 全名：hexo start， 用来表示启动博客的意思。

打开浏览器，搜索：http://localhost:4000

可以在本地查看你的博客效果。

配置GitHub

创建项目代码库

​ 登陆后点击New repository
​ 配置仓库信息如下图所示：

​

注意：仓库名字的格式为 用户名.github.io

将本地的Hexo文件更新到Github的库中

打开自己的项目（用户名.github.io），复制Github项目仓库地址。

修改 hexo 配置文件 _config.yml ：

deploy:
  type: 'git'
  repo: https://github.com/xin-shang/xin-shang.github.io.git
  branch: master

将hexo部署到Github:

$ hexo clean
INFO  Deleted database.
INFO  Deleted public folder.


$ hexo g -d

命令执行后，需要输入Github的账号和密码，输入后，Hexo项目会自动部署到Github上。

操作完成后，我们的博客就完成部署了，我们在浏览器中搜索 用户名.github.io 就可以访问自己的博客了。

其他基础

上传自己的第一篇博客

生成博客 .md 文本：

$ hexo n "你第一篇博客的名字"

$ hexo n " " 全称：hexo name “ “， 表示新建一篇名为“**” 的hexo博客

打开文档写出内容并保存。（关于markdown文档的编辑，我会单独写笔记）

//先清理一下哈~
$ hexo clean

//如果想先在本地预览一下博客效果，用start命令
$ hexo s

//如果没问题或可以直接上传，就用
$ hexo g -d

-d 全称： deploy，部署的意思。

输入Github的用户名和密码，就能成功部署了。
