Git笔记(1) – Git简介

前言

git 是很常用的版本控制系统。做过团队项目的人应该对它不陌生。

前段时间有在学git。这里就是第一篇git笔记。

Git是什么？

Git是一个开源的分布式版本控制系统，初始目的是为了更好的管理__Linux内核__。

后来git内核日渐成熟，已经可以独立地用作版本控制。

与git相似的版本控制系统还有SVN。

Git与SVN

Git 不仅仅是个版本控制系统，它也是个内容管理系统(CMS)，工作管理系统等。

如果你是一个具有使用 SVN 背景的人，你需要做一定的思想转换，来适应 Git 提供的一些概念和特征。

Git 与 SVN 区别点：

  • 1、Git 是分布式的，SVN 不是：

    这是 Git 和其它非分布式的版本控制系统，例如 SVN，CVS 等，最核心的区别。

  • 2、Git 把内容按元数据方式存储，而 SVN 是按文件：

    所有的资源控制系统都是把文件的元信息隐藏在一个类似 .svn、.cvs 等的文件夹里。

  • 3、Git 分支和 SVN 的分支不同：

    分支在 SVN 中一点都不特别，其实它就是版本库中的另外一个目录。

  • 4、Git 没有一个全局的版本号，而 SVN 有：

    目前为止这是跟 SVN 相比 Git 缺少的最大的一个特征。

  • 5、Git 的内容完整性要优于 SVN：

    Git 的内容存储使用的是 SHA-1 哈希算法。这能确保代码内容的完整性，确保在遇到磁盘故障和网络问题时降低对版本库的破坏。

Git库目录

库目录：

  • hooks：存储钩子的文件夹

  • logs：存储日志的文件夹

  • refs：存储指向各个分支的指针（SHA-1标识）文件

  • objects：存放git对象

  • config：存放各种设置文档

  • HEAD：指向当前所在分支的指针文件路径，一般指向refs下的某文件

最常用的Git命令

  • 创建新文件夹，打开

  • 创建新的git仓库

    git init

  • 从远端服务器上克隆项目到本地（如果你的团队成员已经构建好框架）

    git clone <服务器地址>

  • 这时就可以编写代码了，当阶段性代码完成时进入下一步

  • 更改代码（将代码加到暂存区）

    git add .

  • 提交代码到本地库

    git commit -m "提交信息"

  • 提交代码到远程仓库（远端服务器）

    如果你是clone代码到本地修改后，上传：

    git push origin <分支名>

    如果是自己写代码，第一次向远程仓库上传

    git remote add origin <服务器地址>

  • 分支

    // 创建分支_两种方法
git branch <新分支的名字>
git checkout -b <新分支的名字>

//查看分支
git branch

// 切换分支
git checkout <分支名>

// 删除分支
git branch -d <分支名>

// 将分支推送到远程
git push origin <分支名>

  • 拉取 & 合并

    // 拉取远程某分支内容
// 小组中每人每天都在写新内容并上传到远程仓库，所以你需要把它从远程拉取到本地
git pull origin <分支名>

// 把某分支合并到当前分支
// 不同分支的代码会有互补和冲突，需要用这行代码进行检验
git merge <（某）分支名>

// 合并后，上传前不要忘了要add
git add .

// 在合并改动之前，你可以使用下行代码预览差异
git diff <源分支> <目标分支>

  • 更换本地改动

    // 当操作失误时，可用这行代码替换掉本地改动
git checkout <文件名>

// 假如你想丢弃你在本地的所有改动与提交
// 可以到服务器上获取最新的版本历史，并将你本地主分支指向它
git fetch origin
git reset --hard origin/master

  • 其他

    // 内建的图形化git
gitk

// 彩色的git输出
git config color.ui true

// 显示提交记录
git log             //完整记录
git log --oneline    //简易记录

// 交互式添加文件到暂存区
git add -i
